iOS-数据持久化

发表于 2016-04-05   |   分类于 iOS开发   |  

iOS开发中几种常见的数据持久化方法：

  • 属性列表
  • 对象归档
  • SQLite3
  • Core Data

这四种方法都使用iOS的文件系统。

1.属性列表

涉及到的主要类：NSUserDefaults,一般[NSUserDefaults standardUserDefaults]就够用了。

2.对象归档

对象归档需要实现NSCoding协议。NSCoding声明了两个方法都是需要的，一个方法将对象编码到归档中，另一个方法对归档创建新对象，都传递一个NSCoder实例。也可以使用KVC进行编码和解码。

建议对象也同时实现NSCopying协议，该协议允许复制对象，可以使数据模型对象具备较大的灵活性。

3.SQLite3

SQLite3是iOS的嵌入式SQL数据库，使用结构化查询语言。关于SQLite的开发资料较多，建议不直接操作SQLite库，而是采用一些开源的第三方库来进行操作。比如：FMDB对SQLite都做了不错的封装。

4.Core Data

Core Data本质上是使用SQLite保存数据，但是它不需要编写任何SQL语句。要使用Core Data，需要在Xcode中的数据模型编辑器中设计好各个实体以及定义好他们的属性和关系。之后，通过操作这些对象，结合Core Data完成数据的持久化。
