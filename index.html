Objective-C 属性(@property和@synthesize)

发表于 2016-03-26   |   分类于 编程基础   |  

@property是Objective-C的关键词，用来让编译好器自动生成与数据成员同名的方法声明。
@synthesize则是用来生成对应声明方法的实现。

1.property语法格式：

@property (参数1, 参数2) 类型名;

assign: 简单赋值，不更改索引计数，适用于简单数据类型。
retain: 使用引用计数，当引用计数为0时会调用dealloc释放内存。
copy: 适用非共享内存。
atomic: 默认属性，线程安全型变量，只能有一个线程进行访问。非ARC中需要设置访问锁保证正确的get/set。
nonatomic: 非线程安全型变量，多个线程可同时访问，容易失控。
weak: 与assign对应，用于IBOutlets: 

@property (weak,nonatomic) IBOutlet UIButton *myButton;

strong: ARC中默认，等于非ARC中多retain。用于ARC: 

@property (strong,nonatomic)ViewController *viewController;

当strong类型的指针被释放掉之后，所有指向同一对象的weak指针都会被清零。
readonly: 只有get方法，没有set方法。
readwrite: 默认属性，有get/set方法。
unsafe_unretauined: 用于ARC，与assign相似。它只是告诉ARC如何正确地调用声明为unsafe_unretauined变量的retain和release

2.@property的实现方式:

在头文件中声明属性后在实现文件中用关键字@synthesize实现getter，setter。

按照约定赋值方法以set开头，取值方法以实例变量名命名。当使用点语法给存取器发送消息时；get与set的语法是相同的。

总结：@property等同于在.h文件中声明实例变量的get/set方法，@synthesize等同于在.m文件中实现实例变量的get/set方法。使用@property和@synthesize创建存取器要比手动声明两个存取方法（getter和setter）更简单。而且我们在使用属性时可以使用点语法赋值或取值。

iOS开发入门——Objective-C基础语法

发表于 2016-03-20   |   分类于 编程基础   |  

1.文件扩展名

Objective-C是C语言的超集，类似于C语言，Objective-C中用.h表示头文件，源文件可以是以.m, .mm扩展名文件 |

.h文件： 头文件，包含类、类型、函数、常量的声明。
.m文件： 源文件，可以同时包含Objective-C和C代码。
.mm文件：源文件，可以同时包含Objective-C、C和C++代码。

2.类

@class RootViewController;

@interface MyClass : NSObject
{
    UIWindow *window;
    RootViewController    *viewController;
}
//方法声明写在这里
@end

MyClass为类名，NSObject为父类名。中括号中的为成员变量声明。

3.方法

Objective-C中方法可以声明类方法和实例方法。方法声明包括方法类型、返回类型、一个或多个签名关键字、参数类型和参数名称。
-(void)insertObject:(id)anObject atIndex: (NSString)index;
(-)减号表示这是一个实例方法，对应的(+)表示类方法。(void)表示返回类型。冒号用来声明参数，如果没有参数，则只写第一个签名关键字并忽略冒号。
方法的调用：实例方法需要先进行实例化才能调用，类方法可以直接调用。
方法调用需要用到方括号语法： 

[myClass insertObject: anObject atIndex: 0];

以上语句为调用myClass的insertObject方法。

4.属性

属性声明和方法声明一样，也是在类接口定义中进行，用@property表示。 

@property BOOL flag;  
@property NSString *myStr;  
@property UIView *rootView;  
@synthesize flag;  
@synthesize myStr;  
@synthesize rootView;  //只声明getter方法

@synthesize用来生成属性对应的setter和getter方法。

5.协议

协议用来声明能被任何类实现的方法，但本身并不是类。 

@interface MyClass :NSObject<UIApplicationDelegate,AnotherProtocol> { }        
@end

尖括号<>表示该类采用某个协议。协议没有父类，也没有实例变量。协议的声明只有一下一个方法： 

@protocol MyProtocol    
- (void)myProtocolMethod;  
@end
